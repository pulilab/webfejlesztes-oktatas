Gyakorlati óra

Követelmények

  • Heti 1,5 óra gyakorlós elmélet
  • Értékelés:
    • Házi feladatok, hétről, hétre (12*10 pont)
    • Projektmunka (70 pont)
    • Stackowerflow pontok (minden 10 pont SO-n 1 pont a végén)
    • Összesen: 190 pont
    • 60%-ig (114 pont) bukás

Előfeltétel

  • HTML + CSS
  • Programozási alapok

Ha nem szeretnél a webes technológiákban és szkriptnyelvekben mélyre merülni, ez a kurzus nem neked való.

A házi feladatok közt van felkészülés (olvasás, tutorialok teljesítése) és konkrét beadandó programocska is.

Kb heti 3-4 óra munka

Eszközök

  • git
  • github
  • stackoverflow, blogok, levlisták
  • npm, grunt, ... libraries and frameworks
  • sublime text 2, jshint/jslint
  • jsFiddle.net
  • Az eszközök célja, hogy egyszerűsítsék a munkát.
  • Nyílt szabvány, nyílt kód => közösség
  • Menjünk végig minden eszközön:
    1. Git intro (ajánlott: git flow)

    2. GitHub: Mindenki hozzon létre egy fiókot, töltsön fel egy ssh kulcsot, és indítson egy új repo-t egy saját honlapnak. Olvass utána a Github Pages szolgáltatásnak, és indíts egy statikus honlapot, ami ott meg is jelenik. Küldd el a honlap linkjét nekem.

      Rendszeresen kell majd cikkekről rövid összefoglalót írni, amiket ide kell majd kiraknod.
    3. Javascript alapokhoz DailyJS #js101 címkézett cikkei. Gyakorlatilag ezt fogjuk a következő 2 hétben végigvenni.
    4. Stackoverflow: nézegesd, ha kérdésed van ott kérdezz
    5. Sublime Text 2: nézegesd, ismerkedj vele

Egy javaslat

One of the things I recognize at Google is how productive developers surround themselves with powerful tools for iterative development and debugging. For us front-end developers, the ecosystem of tools has exploded in the past two years, as we have a lot more software and libraries beyond Firebug and jQuery to help us build webapps. In the talk below I walk through the current ecosystem of tools and how they can make your development experience a more enjoyable one.
