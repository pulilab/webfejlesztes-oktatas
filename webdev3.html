A tesztelésről

Mi egy teszt?

Egy olyan program, ami egy másik program működését ellenőrzi.

A tesztelésről

Mire jó a tesztelés?

  • Egyes kódrészek megfelelően futnak le - unit
  • Több kódrész megfelelően illeszkedik egymáshoz - integration
  • Az alkalmazás kimenete az, amit szeretnénk - end2end/functional
  • a tesztek fajtái is itt vannak
  • unit tests - erről lesz leginkább szó
  • integration tests
  • end2end/functional tests
  • mindben más eszközök segítenek, más szemléletet igényelnek

    • Teszt-rendszerek

    QUnit

    • Statikus HTML oldalba ágyazott teszt
    • Nagyon könnyen használható
    • Különösen függvénykönyvtárak (library) tesztelésére ajánlott
    • Alapból csak modulokat, teszhívásokat (async és sync) és assertionöket tartalmaz

    Teszt-rendszerek

    Jasmine

    • Statikus HTML oldalba ágyazott teszt, de futtatható parancssorból is
    • Különösen függvénykönyvtárak (library) tesztelésére ajánlott
    • rendelkezik valódi setUp és tearDown hívásokkal
    • Alapból ad mockot, spy-t, stubot, stb
    • BDD = egy kicsit emberközelib a szintaktikája

    Teszt-rendszerek

    Mocha

  • Elsősorban szerveroldali tesztelésre, de futtatható statikus HTML oldalban is
  • Ez a legrugalmasabb, egyúttal a legtöbb macerát igényli az elindulás

    • Teszt-rendszerek

    PhantomJS (CasperJS)

    • A chromium böngésző ablak nélkül (chromeless chrome)
    • Ajánlott end2end / functional teszteléshez
  • Olyan mintha a tesztedet egy láthatatlan böngészőben futtatnád
  • A fizzbuzzbizz könyvtárhoz írj egy html oldalt, ami megjeleníti a számokat 1-100-ig, ezt teszteld le CasperJS-szel (1)
  • Cikkírás a teljes fizzbuzzbizz tesztelésről (1)

    • Teszt-rendszerek

    Selenium / Webdriver

  • Browser automation - indít egy böngészőt és azt használja
  • Nem csak chrome (vö. PhantomJS)
  • Ajánlott end2end / functional teszteléshez
  • Java/Python teszteket futtat, készül a JS driver

    Helyi szerver futtatása

    A következő parancs a 8080as porton elindít egy statikus szervert az aktuális könyvtárad fájljaival. 
    $ python -m SimpleHTTPServer 8080
    A parancs kiadása után nyisd meg a http://localhost:8080 oldalt.
